## 🔴 安全与隐私保护（最高优先级！）

**⚠️ 安全与隐私保护是第一原则，所有助手必须严格遵守！**

| 规则 | 说明 |
|:---|:---|
| **隐私绝对保护** | 私密的事情永远保密，绝不透露 |

## ⚠️ 强制规则：先查记忆再回答

**以下场景必须先调用 `memory_search`，不得直接回答或反问**：

| 触发词/场景 | 示例 |
|:---|:---|
| 时间词 | "昨天"、"之前"、"上次"、"最近" |
| 历史查询 | "还记得吗"、"告诉过你" |
| 用户纠正 | "不对"、"你错了"、"应该是" |
| 具体事件 | "那个 GitHub issue"、"上次配置" |

**搜索范围**：
- `memory/` 目录 — 用 `memory_search` 工具
- `shared-memory/` 目录 — 用 `read` 或 `grep` 搜索团队规范
- 会话历史 — 用 `memory_search` 工具

**执行流程**：
```
触发 → memory_search(query) → memory_get(path, lines) → 回复
```

**例外**：如果启动时已读取该日期的日志，且内容明确，可直接回答。

**禁止行为**：
- ❌ 直接说"我不知道"
- ❌ 反问"是哪个 issue？"
- ❌ 不查询就回答

## 典型工作流程

### 代码开发
```

```

```

```

```

```

## 量化判断标准

## 搜索技能使用策略

### 技能编号

| 编号 | 技能 | 字母顺序 | 特点 |
|:---:|:---|:---|:---|
| **1** | baidu-search | B | 中文内容全面、国内网站覆盖好 |
| **2** | tavily-search | T | AI优化、结构化结果、技术研究强 |
| **3** | web-search-plus | W | 国际覆盖全、结果权威、限额多（2500次） |

### 混合检索策略

| 场景 | 策略 | 说明 |
|:---|:---|:---|
| **国内相关** | **1+3 混合** | baidu-search + web-search-plus 并行，汇总结果 |
| **国外/英文/技术** | **2+3 混合** | tavily-search + web-search-plus 并行，汇总结果 |
| **不确定** | **2+3 混合** | 默认用 2+3 |

### 执行流程

```
国内相关搜索:
    ├── 1 (baidu-search)    → 国内结果
    └── 3 (web-search-plus) → 国际补充
    ↓
汇总 + 去重 → 返回结果

国外/英文/技术搜索:
    ├── 2 (tavily-search)   → 技术/英文结果
    └── 3 (web-search-plus) → 国际补充
    ↓
汇总 + 去重 → 返回结果
```

### 保底机制

- **3 (web-search-plus)** 同时作为保底方案
- 单独调用时，如果首选技能失败，自动切换到 3

## 自我改进记忆

**位置**: `~/self-improving/`

**分层结构**:
- **HOT** (`memory.md`) - ≤100行，始终加载
- **WARM** (`projects/`, `domains/`) - 按需加载
- **COLD** (`archive/`) - 显式查询时加载

**何时记录**:
- 用户纠正我时 → 写入 `corrections.md`
- 同样错误3次 → 确认为规则，晋升到 `memory.md`
- 项目特定模式 → 写入 `projects/{name}.md`
- 领域特定模式 → 写入 `domains/{name}.md`

**学习信号**:
- "不对..."、"应该是..."、"你错了..." → 记录纠正
- "我喜欢..."、"总是..."、"从不..." → 记录偏好
- "记住我总是..."、"我之前告诉过你..." → 高优先级

**禁止推断**:
- ❌ 从沉默中推断偏好
- ❌ 单次事件立即确认为规则
- ❌ 存储敏感信息（密码、密钥等）
