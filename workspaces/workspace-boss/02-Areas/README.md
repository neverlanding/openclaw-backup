# 02-Areas - 责任领域目录

> **这是什么**：需要**长期维护**的责任和工作领域（没有 deadline）
> **什么时候用**：定义工作规范、维护核心配置、建立长期流程
> **存放内容**：身份定义、工作规范、核心信念、工具使用说明
> **维护方式**：定期回顾、持续优化、长期积累

**与 Projects 的区别**：Areas 是长期职责，Projects 是阶段目标

## 领域 vs 项目

| | Areas（领域） | Projects（项目） |
|:---|:---|:---|
| **时间** | 长期持续 | 有截止日期 |
| **目标** | 维护状态 | 达成目标 |
| **示例** | 系统运维、多助手协调 | 8-Agent建设、配置优化 |
| **更新频率** | 低频（按需调整） | 高频（随项目推进） |
