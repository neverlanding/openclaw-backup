## Memory（记忆管理）

### 记忆分层

| 层级 | 位置 | 用途 |
|:---|:---|:---|
| 日常笔记 | `memory/YYYY-MM-DD.md` | 当日事件、上下文、决策 |
| 长期记忆 | `MEMORY.md` | 项目看板、待办、决策记录 |
| 自我改进 | `~/self-improving/` | 执行改进、偏好、工作流模式 |

### Self-Improving 模式

当前模式：**Passive**（被动学习）

| 模式 | 行为 |
|:---|:---|
| Passive | 仅从明确的纠正中学习 |
| Active | 3次重复后建议模式 |
| Strict | 每条记录都需要确认 |

### 任务前检查

执行非简单任务前：
1. 读取 `~/self-improving/memory.md`
2. 列出可用文件：
   ```bash
   for d in ~/self-improving/domains ~/self-improving/projects; do
     [ -d "$d" ] && find "$d" -maxdepth 1 -type f -name "*.md"
   done | sort
   ```
3. 读取最多 3 个匹配的 domain 文件
4. 如有明确活跃项目，读取 `~/self-improving/projects/<project>.md`

### 记录规则

| 触发 | 动作 |
|:---|:---|
| 用户明确纠正 → 写入 `~/self-improving/corrections.md` |
| 可复用的全局规则/偏好 → 写入 `~/self-improving/memory.md` |
| 领域特定的教训 → 写入 `~/self-improving/domains/<domain>.md` |
| 项目专属覆盖 → 写入 `~/self-improving/projects/<project>.md` |

- 保持条目简短、具体、一条教训一个要点
- 如范围不明确，默认使用 domain 而非 global
- 纠正或强可复用教训后，在最终回复前写入
